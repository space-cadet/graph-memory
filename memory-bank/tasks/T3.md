# T3: Memory Search Bridge

**Status:** ⏳ pending  
**Priority:** HIGH  
**Depends on:** T2  
**Started:** 2026-06-18  

## Goal

Create a `search-graph.cjs` wrapper that the agent can call to query the graph as a memory source. Takes a natural language query, returns structured results.

## Problem

The graph is queryable via `knowledge-graph.cjs` CLI, but the agent has no way to use it during conversations. When you ask "what were we working on last week?" the agent reads MEMORY.md and session files — it never queries the graph.

## Solution

A bridge script that:

1. Takes a search query (e.g., "chimera project", "QHE paper", "cron job failures")
2. Searches graph entities (fuzzy match on name + type)
3. For each match, fetches related entities (neighborhood)
4. Finds paths between entities if multiple matches
5. Returns structured JSON: entities, relationships, context, dates

## Example Flow

**User:** "What did we decide about Chimera?"
**Agent → search-graph:** `search-graph "chimera project decisions"`
**search-graph → returns:**
```json
{
  "query": "chimera project decisions",
  "entities": [
    {"name": "chimera-chat", "type": "project", "mentions": 4},
    {"name": "arxivite", "type": "project", "mentions": 12}
  ],
  "relationships": [
    {"source": "chimera-chat", "target": "arxivite", "type": "uses_feature"},
    {"source": "chimera-chat", "target": "obsidian-ai", "type": "uses_feature"}
  ],
  "context": [
    "2026-06-12: Decided to merge LaTeX rendering from arxivite with tool calling from obsidian-ai",
    "2026-06-15: Chose Chart.js for visualization, not D3.js"
  ]
}
```

## Implementation

1. Create `scripts/search-graph.cjs`
2. Query graph entities with fuzzy matching
3. Traverse relationships (1-2 hops)
4. Extract temporal context from session dates
5. Return JSON that agent can parse into natural language

## Notes

The graph currently has temporal info via `date:YYYY-MM-DD` session entities, but not actual decision/context text. We may need to store snippet context in relationship table or add a new table for "conversation snippets".
