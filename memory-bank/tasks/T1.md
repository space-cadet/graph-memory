## Implementation Details

### Heartbeat Protocol (HEARTBEAT.md)

**Rotation:** 3rd heartbeat, every-2nd (base cadence)  
**Self-modifying:** YES — de-escalates if no new sessions, terminates if graph disabled

**Activation:** `graph-memory` repo exists AND `session-entity-extractor.cjs` exists
**Base cadence:** Rotation 3, every-2nd heartbeat only
**Escalation:** New session files detected AND graph hasn't been updated in >2 hours
**De-escalation:** No new session files for 10 consecutive heartbeats → once per day
**Termination:** User says "disable graph memory" OR repo removed
**Idempotency:** Check `.session-watermark.json` `lastProcessedMtime` vs. newest session file mtime

**Command:**
```bash
cd ~/.openclaw/workspace/code/graph-memory
node scripts/session-entity-extractor.cjs 2>&1 | tail -5
```

### Watermark System

- **Location:** `~/.openclaw/workspace/.openclaw_memory/.session-watermark.json`
- **Fields:** `lastProcessedMtime`, `processedCount`, `lastRun`
- **Logic:** Only processes files with `mtime > lastProcessedMtime`
- **State:** `{"lastProcessedMtime":1781811176228,"processedCount":6349,"lastRun":"2026-06-18T19:30:00Z"}`

## Notes

The session extractor runs incrementally using the watermark. It does NOT do full rebuilds in heartbeat mode. If a full rebuild is ever needed, the user must run `node session-entity-extractor.cjs --all` manually.

The graph.db stays in `~/.openclaw/workspace/.openclaw_memory/` (shared with the visualization system). The repo at `code/graph-memory/` contains the scripts and memory bank but references the shared database.
