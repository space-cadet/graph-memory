# T1: Graph Update Automation

**Status:** 🔄 in_progress  
**Priority:** HIGH  
**Started:** 2026-06-18  

## Goal

Integrate graph build into the heartbeat rotation so the graph stays current with new sessions. Run every 2nd heartbeat (~1 hour) alongside journal processing.

## Current State

- `build-graph.cjs` exists but is **not automated**
- Last manual build: 2026-06-18 02:27 UTC (stale by ~16 hours)
- `batch-process-all-sessions.sh` runs in heartbeat but only generates text journals, not graph updates
- `build-graph.cjs` does **full rebuilds** — no incremental mode despite `--incremental` flag

## Approach

1. Add `build-graph.cjs` call to heartbeat rotation (every 2nd heartbeat)
2. Fix `--incremental` flag to actually work (track last-processed session files)
3. Use `session-entity-extractor.cjs` (T2) instead of journal-based extractor

## Notes

The heartbeat already processes sessions → journals. The gap is journal → graph. We should either:
- Run `build-graph.cjs` after `batch-process-all-sessions.sh` in the same heartbeat, OR
- Run it on the next heartbeat (every 2nd, since journal processing is also every 2nd)

Given the full-rebuild cost, Option B (every 2nd) is safer until incremental mode works.
