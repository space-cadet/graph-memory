# T5: Historical Backfill

**Status:** ⏳ pending  
**Priority:** LOW  
**Depends on:** T2  
**Started:** 2026-06-18  

## Goal

Reprocess all historical session JSONLs with the new `session-entity-extractor.cjs` to rebuild the graph from full conversation text, not just recent journals.

## Scope

- **13,702 session JSONL files** in `~/.openclaw/agents/main/sessions/`
- Currently only **25 journal files** have been processed into the graph
- This means most of the conversation history (May-June 2026) is in the graph as `error` and `file` entities, not meaningful concepts

## Approach

1. Run `session-entity-extractor.cjs` with **no watermark** (process all sessions)
2. Batch process: 100 sessions at a time to avoid memory issues
3. Compare new graph stats to old graph stats
4. Verify quality: search for known topics (QHE, Chimera, CJP, etc.)

## Risks

- **Time**: 13K sessions could take hours to process
- **Memory**: Full text extraction from all sessions = large in-memory dataset
- **Quality**: Even with full text, the regex patterns might still miss concepts

## Mitigation

- Batch in chunks of 100 sessions
- Use `--date` range to test on a subset first (e.g., last 7 days)
- Validate before full backfill
