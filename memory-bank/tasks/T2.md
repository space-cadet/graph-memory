## Goal

Create a new extractor that reads **raw session JSONL files** directly and extracts entities from the **full conversation text**, not truncated journal summaries.

## Problem

The current `entity-extractor.cjs` reads `journal/YYYY-MM-DD.md` files. These journals are generated by `journal-writer.cjs` which:
- Truncates user messages to **100 characters**
- Truncates assistant thinking to **300 characters**
- Filters out "system messages" (which includes most of your actual requests)
- Only logs tool calls and errors

So when you search for "chimera" in the graph, you get **0 results** â€” because the word "chimera" might appear in a 500-character message that got truncated to 100 chars, or was classified as a system message and skipped entirely.

## Solution

A new script `session-entity-extractor.cjs` that:

1. Reads raw `.jsonl` files from `~/.openclaw/agents/main/sessions/`
2. Extracts `user` role messages with **full text** (no truncation)
3. Extracts `assistant` role messages with **full text**
4. Runs the same regex patterns from `entity-extractor.cjs` on the complete text
5. Uses the same SQLite schema and canonicalization rules
6. Tracks a watermark (which session files / lines already processed)
7. Writes to `graph.db` incrementally

## Implementation

**Created:** `scripts/session-entity-extractor.cjs`

### Key Features
- **Watermark tracking**: `.session-watermark.json` stores `lastProcessedMtime`, `processedCount`, `lastRun`
- **Incremental processing**: Only processes files with `mtime > watermark.lastProcessedMtime`
- **Full text extraction**: User + assistant messages + thinking blocks (no truncation)
- **Content array parsing**: Handles JSONL content format `[{type:"text", text:"..."}, {type:"thinking", thinking:"..."}]`
- **Same entity patterns**: Reuses regex from `entity-extractor.cjs` but fed richer input
- **Same schema**: Writes to `graph.db` entities/relationships tables

### CLI Options
```bash
node session-entity-extractor.cjs                    # process all new sessions
node session-entity-extractor.cjs --all              # force full rebuild
node session-entity-extractor.cjs --dry-run          # print, don't write
node session-entity-extractor.cjs --file <path>      # process specific file
```
