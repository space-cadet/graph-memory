## Goal

Add vector embeddings to enable semantic search across sessions and entities.

## Problem Statement

Current search is literal string matching:
- "chat component" does NOT find "chimera-chat"
- "QHE paper" does NOT find "arxiv:2605.15200"
- "the tool we built for Obsidian" finds nothing

## Solution

### Embedding Model

| Model | Dimensions | Speed | Quality | Size |
|-------|-----------|-------|---------|------|
| all-MiniLM-L6-v2 | 384 | Fast | Good | 80MB |
| all-mpnet-base-v2 | 768 | Medium | Better | 400MB |

**Recommendation:** Start with all-MiniLM-L6-v2 (384-dim). Upgrade if quality insufficient.

### Schema

```sql
CREATE TABLE embeddings (
  id INTEGER PRIMARY KEY,
  entity_name TEXT,
  content TEXT,
  embedding BLOB,  -- 384 × float32 = 1536 bytes
  created_at TEXT
);

CREATE INDEX idx_embeddings_entity ON embeddings(entity_name);
```

### Storage Estimate

- Per embedding: 1,536 bytes (384 × 4 bytes)
- 13K sessions: ~20MB
- 100K entities: ~150MB
- Total: < 200MB — trivial

### Search Implementation

```javascript
// Cosine similarity search
function findSimilar(query, topK = 10) {
  const queryEmbedding = embed(query);
  const candidates = db.prepare(`
    SELECT entity_name, content,
           (embedding * ?) / (|embedding| * |query|) AS similarity
    FROM embeddings
    ORDER BY similarity DESC
    LIMIT ?
  `).all(queryEmbedding, topK);
  return candidates;
}
```
