# T4: Entity Quality Improvements

**Status:** ⏳ pending  
**Priority:** MEDIUM  
**Depends on:** T2  
**Started:** 2026-06-18  

## Goal

Improve the quality and diversity of extracted entities. Currently 90% of entities are files, errors, and arXiv IDs. Missing: projects, concepts, decisions, topics, questions.

## Current Stats

| Type | Count | % |
|------|-------|---|
| file | 513 | 58% |
| research_paper | 216 | 25% |
| error | 74 | 8% |
| project | 41 | 5% |
| tool | 16 | 2% |
| person | 14 | 2% |
| institution | 6 | <1% |
| concept | 1 | <1% |

## Problems

1. **Concept count = 1** — We discuss quantum computing, LQG, braids, TQC every day. Only 1 concept entity means the regex is missing almost everything.
2. **No decision entities** — "Decided to use k2.7-code", "Chose Chart.js over D3" — these are lost
3. **No topic/question entities** — "How do we integrate LLM backend?" — lost
4. **Error pollution** — 74 error entities from cron job failures, not real work
5. **Relationship types are weak** — 91% are `mentioned_with` (co-occurrence), only 9% are meaningful

## Improvements

### New Entity Types
- `decision` — explicit choices made in conversations
- `topic` — discussion subjects
- `question` — unresolved questions
- `goal` — stated objectives

### Better Extraction Patterns
- `decision`: /decided to|chose|went with|settled on|agreed on/i
- `topic`: /discussing|talking about|working on|focusing on/i
- `question`: /how do we|what if|should we|can we/i

### Richer Relationships
- `decided_to_use` (project → tool)
- `implements` (project → concept)
- `answers` (project → question)
- `depends_on` (project → project)
- `replaces` (project → project)
