# T8: Background Processing Queue

**Status:** ⏳ pending  
**Priority:** 🟡 MEDIUM  
**Started:** 2026-06-19  
**Phase:** 3 of 6

## Goal

Decouple graph extraction from heartbeat to prevent blocking.

## Problem

Current: Heartbeat runs `session-entity-extractor.cjs` synchronously.
- At 13K sessions, this blocks for minutes
- SQLite is single-writer — queries blocked during extraction

## Solution

```
Heartbeat (every 30 min):
  1. Check newest session mtime vs watermark
  2. If new sessions exist: enqueue session IDs
  3. Done (< 100ms)

Worker (every 5 min):
  1. Dequeue up to N sessions
  2. Run LLM extraction
  3. Update graph
  4. Update watermark
```

## Implementation

```bash
# Queue file
~/.openclaw_memory/.extraction-queue  # JSON array

# Worker cron
*/5 * * * * cd ~/code/graph-memory && node scripts/extraction-worker.cjs
```
